LEARNING PRACTICE FLOW – PARTIAL SPEC (EXTENSION)

Goal
Расширить существующий экран работы с текстом, добавив пошаговый флоу для перехода от понимания текста к активному воспроизведению (письму).

Экран деталей текста уже существует и не пересматривается.
Новые изменения затрагивают только организацию флоу и дополнительные режимы взаимодействия.

HIGH-LEVEL FLOW CONCEPT

Работа с одним и тем же текстом происходит в нескольких последовательных режимах.

Ментальная модель:

один текст

один экран

несколько способов взаимодействия с этим текстом

Переход между режимами должен быть:

явным

контролируемым пользователем

обратимым

FLOW ORGANIZATION

Для организации флоу используется переключение режимов
(табы или эквивалентный компонент).

Каждый режим:

использует тот же текст

не меняет навигацию (sidebar остаётся прежним)

не требует перезагрузки данных

Переключение режима меняет:

представление текста

тип взаимодействия пользователя с текстом

PRACTICE MODES (ORDERED BY DIFFICULTY)

Mode 1 — Read
Существующий режим.
Используется как база и точка возврата.

Mode 2 — Fill (Gap-fill, drag & drop)

Текст содержит пропуски.

Пользователь восстанавливает текст, перетаскивая слова.

Цель: закрепление формы слов и узнавание.

Mode 3 — Order (Sentence order)

Предложения обрабатываются по одному.

Слова в предложении перемешаны.

Пользователь восстанавливает правильный порядок.

Цель: понимание структуры и синтаксиса.

Mode 4 — Write (Gap typing)

В тексте присутствуют пропуски.

Нет списка слов и перетаскивания.

Пользователь вводит слова вручную.

Цель: активное воспроизведение формы и орфографии.

COMMON RULES FOR ALL PRACTICE MODES

Исходный текст всегда один и тот же.

Упражнения генерируются динамически из текста.

Выполнение упражнений не требует сетевых запросов или LLM.

Проверка результатов:

по явному действию пользователя (например, кнопка “Check”)

строгая (правильно / неправильно).

Нет:

прогресса

баллов

подсказок

объяснений ошибок

FLOW INTENTION

Флоу выстроен как постепенное усложнение:

понимание → узнавание → структура → воспроизведение

Пользователь может:

оставаться в одном режиме

переходить между режимами в любом порядке

возвращаться к Read в любой момент
